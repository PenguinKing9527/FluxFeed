# 功能设计：文章阅读

## 功能概述

文章阅读是用户使用频率最高的核心功能。插件提供两种阅读方式：**内置阅读视图**（沉浸式，提取正文渲染）和**跳转原文**（打开新标签页）。内置阅读视图强调排版质量和阅读舒适度。

---

## 用户故事

- 作为用户，我希望在插件内直接阅读文章，不需要跳转到原网站
- 作为用户，我希望阅读视图清洁、无广告，专注于正文内容
- 作为用户，我希望可以调整字体大小和明暗主题
- 作为用户，我希望滚动文章时自动标记为已读

---

## 阅读视图设计

### 正文内容处理

Feed 正文 HTML 经过以下处理后渲染：

1. **HTML 净化**（防 XSS）
   - 白名单标签：`p, h1-h6, ul, ol, li, blockquote, pre, code, img, a, strong, em, figure, figcaption, table, tr, td, th`
   - 移除所有 `style`, `class`, `id` 属性（通过自定义 CSS 统一排版）
   - 移除 `script`, `iframe`, `form`, `input` 等危险标签

2. **图片懒加载**
   - 为所有 `<img>` 添加 `loading="lazy"`
   - 图片点击可全屏预览

3. **链接处理**
   - 所有链接添加 `target="_blank" rel="noopener noreferrer"`
   - 链接点击在新标签页打开

4. **代码块**
   - `<pre><code>` 块添加横向滚动 + 等宽字体
   - 支持一键复制代码

5. **内容回退**
   - 若 Feed 仅提供摘要（`<description>` 内容 < 500 字），自动显示"查看原文"提示

### 列表排序与过滤

| 选项 | 说明 |
|---|---|
| 全部 / 仅未读 | 全局过滤开关（持久化到设置） |
| 时间倒序 | 默认排序（最新在前） |
| 时间正序 | 可切换 |
| 按 Feed 分组 | 在 Feed 视图下，仅显示该 Feed 文章 |

### 虚拟滚动

- 文章列表使用虚拟滚动（Virtual List），只渲染可视区域内的条目
- 支持 10,000+ 条文章无卡顿
- 实现方案：使用 `@tanstack/react-virtual`（轻量、无样式、与 React 18 兼容）
  - Popup 窗口宽度固定，列表项高度可预估（约 88px），适合 fixed-size 模式
  - 无需引入 react-virtualized（API 过时）或 react-window（功能受限）

## 已读状态管理

| 触发动作 | 行为 |
|---|---|
| 打开文章阅读视图 | 若设置"打开即已读"开启，标记已读 |
| 滚动超过文章 80% | 自动标记已读（默认行为） |
| 点击列表项旁边的圆点 | 手动切换 已读 / 未读 |
| 右键菜单 → 标记已读 | 在列表中批量标记 |
| 工具栏"全部标记已读" | 清空当前视图的未读 |

## 键盘快捷键

| 快捷键 | 功能 |
|---|---|
| `j` / `↓` | 下一篇文章 |
| `k` / `↑` | 上一篇文章 |
| `o` / `Enter` | 打开原文（新标签页） |
| `m` | 切换已读 / 未读 |
| `s` | 收藏 / 取消收藏 |
| `Esc` | 返回文章列表 |
| `r` | 刷新当前 Feed |

## 主题与字体设置

### 主题

| 主题 | 说明 |
|---|---|
| 浅色 | 白底深字，默认 |
| 深色 | 深底浅字，护眼 |
| 跟随系统 | 检测 `prefers-color-scheme` |

### 字体大小

提供 Small / Medium / Large 三档，对应正文字号 14px / 16px / 18px，行高自动适配。

### 字体族

- 正文：衬线字体（Source Serif / Noto Serif CJK），提升中文阅读舒适度
- 代码：等宽字体（JetBrains Mono / Fira Code）
