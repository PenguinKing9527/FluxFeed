# 功能设计：收藏（稍后读）

## 功能概述

收藏功能让用户将感兴趣但暂时没时间细读的文章保存起来，形成个人精选阅读列表。收藏的文章永久保留，不受"每 Feed 最大文章数"和"保留天数"限制。

---

## 用户故事

- 作为用户，我希望快速收藏一篇文章，之后再读
- 作为用户，我希望收藏的文章不会被自动清理
- 作为用户，我希望有一个统一的入口查看所有收藏文章
- 作为用户，我希望取消收藏操作简单，不需要确认弹窗

---

## 收藏入口

收藏操作在以下位置均可触发，保证随手可及：

| 位置 | 交互方式 |
|---|---|
| 文章列表项 | 悬停后出现 ⭐ 图标，点击收藏 |
| 阅读视图顶部工具栏 | 常驻 ⭐ 图标，当前状态高亮显示 |
| 键盘快捷键 | `s` 键切换收藏 / 取消收藏 |
| 列表项右键菜单 | "收藏文章" / "取消收藏" |

收藏/取消收藏均**即时生效**，无需确认弹窗，支持撤销（操作后 3 秒内显示 Toast + "撤销"按钮）。

---

## 收藏列表视图


### 收藏列表特性

- 默认按**收藏时间倒序**排列（最近收藏的在最前）
- 可切换为按**发布时间**排序
- 支持按 Feed 来源过滤（在左侧 Feed 列表中点击某 Feed 时，收藏 Tab 下仅显示该 Feed 的收藏文章）
- 收藏文章数量显示在 Tab 标签旁：`⭐ 收藏 (12)`

### 列表项样式

与普通文章列表一致，额外在右下角显示**收藏时间**（而非发布时间），便于用户回忆收藏顺序。

---

## 数据设计

收藏状态存储在 `articles` 表的 `isStarred` 字段，无需独立表：

```typescript
interface Article {
  // ...
  isStarred: boolean;   // 收藏状态（索引）
  starredAt?: number;   // 收藏时间戳（用于排序）
}
```

收藏查询：

```typescript
// 获取所有收藏文章，按收藏时间倒序
const starred = await db.articles
  .where('isStarred').equals(1)
  .sortBy('starredAt')
  .then(arr => arr.reverse());
```

---

## 数据保护规则

收藏文章不受以下清理策略影响：

- ✅ 免于"每 Feed 最大文章数"限制（超出时优先删除未收藏的已读文章）
- ✅ 免于"保留已读文章天数"自动清理
- ✅ 删除 Feed 时，提示用户"该 Feed 有 N 篇已收藏文章，是否一并删除？"（默认保留）

---

## 与标签的关系

收藏是**轻量的一键操作**，适合"先标记再读"的场景；文章标签是**主动分类**，适合读后整理归档。两者互不替代，可以同时使用：

- 收藏：快速、临时性标记，类似 Inbox
- 标签：读后手动打标，类似归档分类（"值得二读"、"技术参考"、"行业洞察"）
